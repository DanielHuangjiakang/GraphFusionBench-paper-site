Overview

Abstract

GraphFusionBench is a benchmark construction effort for PyTorch Inductor graph fusion research. It aggregates runnable model fragments from nine public benchmarks, standardizes them into single-file CUDA-ready PyTorch modules, and applies AST-level deduplication to obtain a clean sample pool.

From that pool, 10,000 synthetic benchmark samples are generated across eight node-count buckets and 25 template types — covering pointwise chains, residual blocks, LayerNorm, Conv+BN, and attention softmax decomposition. Each sample is compiled with PyTorch Inductor on CUDA and archived with both pre-fusion and post-fusion scheduler dumps.

The resulting artifacts support node-count stratification, fused/non-fused analysis, and instruction-tuning data generation where the input is the pre-fusion graph and the target output follows the baseline post-fusion scheduler result.

Method

Construction Pipeline

01

Build Sample Pool

Collect runnable model fragments from nine public benchmarks (excluding TorchBench). Standardize each as a standalone .py module with Model(nn.Module), get_model(), and CUDA-ready inputs. Remove duplicates via normalized AST signature matching — reducing 40,459 raw files to 19,550 unique samples.

02

Generate & Dump

Synthesize 10,000 benchmark samples across 8 node-count buckets using 25 parameterized template types, targeting realistic fusion ratios per bucket. Compile each with torch.compile(backend="inductor") on CUDA and archive pre-fusion scheduler graphs plus post-fusion baseline results.

03

Construct SFT Data

Sample by bucket and fusion ratio; build Alpaca-format instruction-tuning records where the input is the pre-fusion graph_adj.txt and the output is a reasoning trace derived from the baseline post-fusion dump — covering both fused and non-fused graph patterns.

Training Data

SFT Data Construction

The SFT format follows the Alpaca schema. The instruction is shared across examples: the input is the pre-fusion graph_adj.txt (scheduler node adjacency listing), and the output is a reasoning trace plus JSONL fusion groups derived from the baseline post-fusion dump.

Sampling is stratified by scheduler node count. Buckets B3–B7 target a roughly 1:3 no-fusion to has-fusion split; B0 forces no-fusion (single-node graphs), and B1–B2 use lower fusion ratios (0% / 12.5% / 45%) for structural diversity. Each accepted has-fusion sample is validated against the actual Inductor baseline result — template intent alone does not qualify.

Total: 1,893 SFT records from 10,000 generated benchmark samples.
