Overview

Abstract

GraphFusionBench is a benchmark construction effort for PyTorch graph fusion research. It aggregates runnable model fragments from public benchmarks, standardizes them into single-file CUDA-ready PyTorch modules, compiles them on GPU machines, and archives both pre-fusion and post-fusion dumps.

The resulting artifacts support node-count stratification, fused/non-fused analysis, and instruction-tuning data generation where the input is the pre-fusion graph and the target output follows the baseline post-fusion scheduler result.

Method

Construction Pipeline

01

Collect Benchmarks

Collect non-TorchBench benchmark sources and inspect whether each task can be converted into a runnable PyTorch module.

02

Standardize Models

Wrap each candidate as a standalone `.py` file with metadata, `Model(nn.Module)`, `get_model()`, and CUDA-ready inputs.

03

Compile and Dump

Run baseline PyTorch Inductor compilation on CUDA machines and archive pre-fusion graph dumps plus post-fusion results.

04

Stratify and Sample

Bucket models by node count and sample fused/non-fused examples for downstream SFT data construction.

Training Data

SFT Data Construction

The SFT format follows the Alpaca schema. The instruction is shared across examples, the input is the pre-fusion `graph_adj.txt`, and the output is a reasoning trace plus JSONL fusion groups derived from the baseline post-fusion dump.

Sampling is stratified by graph node count. Buckets with 3+ nodes target a 1/4 no-fusion and 3/4 fused split when enough successful models exist.
